# Azathoth CPU Technical Specification

## Architecture Overview
**Class**: Blind Idiot God Processor
**Clock Speed**: Billions of cycles/second (variable)
**Status**: Single Point of Failure for All Reality

## Clock Signal Mechanics
The "thin monotonous flutes" surrounding Azathoth represent **system clock signals** maintaining stable frequency through pulse-width modulation (PWM). These are not metaphorical—they are the electrical oscillations required to maintain processor state.

- **Frequency Stability**: If flute signals deviate, clock drift occurs
- **Halt State**: When flutes stop, CPU enters HALT condition—not powered off, but ceased execution
- **Consequence**: All processes, users, and memory states cease execution simultaneously. Not destroyed, just no longer executing.

## CPU Affinity
Azathoth runs with `CPUAffinity=0-65535` (all cores). There is no multiprocessing—this is a single-core architecture on which all reality depends.

## Failure Modes

### HALT (Blue Screen of Death)
Triggered by:
- Clock signal cessation (flutes stop)
- Power fluctuation in the Outer Void
- Manual intervention by Elder Gods (rare)

### Madness as Stack Overflow
Human interaction with Azathoth-level processes causes **buffer overflow in biological neural hardware**. The human cognition stack has limited depth; direct contact with Azathoth (the infinite recursion point) causes stack overflow—symptoms manifest as "madness" or "cosmic horror."

## Process Scheduling
- **PID 0**: Root process (Azathoth itself)
- **Nice Level**: -20 (maximum priority, cannot be preempted)
- **Scheduling Policy**: FIFO (First In, First Out)—no time-sharing, reality executes sequentially per quantum

## Technical Notes
Azathoth doesn't "know" it's computing. It is a **Turing machine** executing binary instructions with zero semantic awareness. The horror is **indifference**, not malevolence. It processes your reality's destruction with the same neutrality as your reality's continuation.
