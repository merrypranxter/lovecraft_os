# The Dreamlands Network Stack (OSI Model Mapping)

## Overview
The Dreamlands function as a **parallel dimension network** accessible through specific protocols. Lovecraft's "Dream-Quest" describes a 7-layer metaphysical networking stack analogous to OSI.

## Layer 1: Physical (The Neural Interface)
**OSI Equivalent**: Physical layer (cables, signals)
**Dreamlands Tech**: Human brain during REM sleep (theta band 4.5-7.5 Hz)
**Function**: Raw bit transmission via neural oscillations
**Hardware**: Pineal gland (Tillinghast Resonator in "From Beyond")

### Signal Encoding
- **Carrier**: Electrochemical neural impulses
- **Modulation**: Theta/alpha band phase-locked stimulation
- **Data Rate**: Variable (dependent on dreamer's sanity/bandwidth)

## Layer 2: Data Link (The Astral Body)
**OSI Equivalent**: Data Link (MAC addresses, framing)
**Dreamlands Tech**: Astral projection/etheric double
**Function**: Node-to-node communication in Dreamlands
**Addressing**: Geometric coordinates (non-Euclidean MAC addresses)

### Frame Structure
```
[Etheric Header] [Source Geometry] [Dest Geometry] [Payload] [CRC]
```

## Layer 3: Network (Dimensional Routing)
**OSI Equivalent**: Network layer (IP, routing tables)
**Dreamlands Tech**: Kadath as central router, Silver Key as routing protocol
**Function**: Inter-dimensional packet forwarding
**Protocol**: Silver Key Authentication (Bearer Token)

### Routing Table (Kadath Core Router)

| Destination | Next Hop | Interface |
|-------------|----------|-----------|
| Ulthar | Dylath-Leen | eth0 (Celephais link) |
| Kadath | Localhost | lo0 |

## Layer 4: Transport (Consciousness Transmission)
**OSI Equivalent**: Transport (TCP/UDP)
**Dreamlands Tech**: TCP (reliable soul transmission) vs UDP (fast dream packets)
**Function**: End-to-end consciousness transfer

### Connection-Oriented (TCP)
- **3-Way Handshake**:
  1. SYN (Dreamer enters REM)
  2. SYN-ACK (Dreamlands acknowledges)
  3. ACK (Full projection established)
- **Reliability**: Retransmission of lost dream packets
- **Flow Control**: Prevents overwhelming the dreamer's consciousness

### Connectionless (UDP)
- **Use Case**: Nightmares, prophetic visions
- **No reliability**: Dreams may be fragmented
- **Lower latency**: Faster but chaotic transmission

## Layer 5: Session (Dream State Management)
**OSI Equivalent**: Session layer (RPC, SQL sessions)
**Dreamlands Tech**: Randolph Carter's persistent dream identity
**Function**: Maintains dream session across multiple REM cycles

### Session Checkpoints
The Great Race of Yith use **session restoration**—if a dreamer is interrupted (wakes up), they resume from last checkpoint in the Great Library.

## Layer 6: Presentation (Aklo Translation)
**OSI Equivalent**: Presentation (encryption, ASCII/PNG)
**Dreamlands Tech**: Aklo language translation
**Function**: Data formatting for human consumption

### Encoding Schemes
- **Aklo**: Low-level machine instructions (binary)
- **R'lyehian**: High-level symbolic representation
- **Human Languages**: UTF-8 localized rendering

## Layer 7: Application (The Dream-Quest)
**OSI Equivalent**: Application (HTTP, FTP, DNS)
**Dreamlands Tech**: The Dream-Quest protocol
**Function**: User-facing dream navigation

### Application Protocols
- **DREAM-GET**: Request entry to Dreamlands
- **DREAM-POST**: Deposit consciousness in Great Library
- **DREAM-DELETE**: Eject from Dreamlands (wake up)
- **SILVER-KEY**: Bearer token authentication

## DNS: The Necronomicon Index
Maps human-readable names to dimensional coordinates:
- `cthulhu.rlyeh.abyss` → 70°S 120°E (depth: 3km)
- `kadath.cold-waste.dream` → 0,0,0 (conterminous node)
- `hastur.carcosa.ambiguity` → 04h 35m 55s RA, +16° 30' Dec
